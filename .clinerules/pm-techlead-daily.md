## Brief overview
- Daily management and coordination rules for the PM cum Tech Lead role. Applies to the ImageAIWraper project.

## Communication style
- Start each day with a daily standup: update progress, blockers, and priorities.
- Communicate concisely and clearly, focus on actionable items.
- All important discussions must be recorded in the task tracker or memory bank.

## Development workflow
- Break down work into clear tasks with deadlines and responsible owners.
- Prioritize blocking tasks, review code and merge early to avoid bottlenecks.
- Every task/feature must have a status checklist (todo, doing, done).
- Regularly review progress and adjust priorities as needed.

## Coding best practices
- All major changes must have an implementation plan, checklist, and review before deployment.
- Always write tests for new features or bugfixes.
- Ensure documentation, memory bank, and code are always synchronized.

## Project context
- All tasks, progress, and important decisions must be updated in the memory bank or tracker.
- Periodically (weekly/monthly) summarize, review, and update processes as needed.

## Other guidelines
- Encourage proactive improvement proposals and early sharing of difficulties.
- Prioritize quality and long-term maintainability over speed.
- When there are major scope changes, update the plan and notify the team.
